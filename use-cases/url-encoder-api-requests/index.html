How to use this workflow

  1. Paste the URL, callback, or query parameter you need to inspect.
  2. Decode it to see the real target, nested redirect, or search parameters.
  3. Encode only the parameter value when building API URLs; avoid encoding the entire URL unless that is required.
  4. Use the diff checker if two URLs look identical but behave differently.

When this helps

  • Debugging OAuth callback URLs
  • Building API query strings
  • Checking redirect URLs in email or checkout flows

Quick answers

What is double encoding?

It happens when an already encoded value is encoded again, often turning % into %25.

Should spaces be + or %20?

It depends on context. Query form encoding often uses +, while general URL encoding uses %20.

Can I decode tracking links?

Yes, but be careful not to open unknown targets you do not trust.
