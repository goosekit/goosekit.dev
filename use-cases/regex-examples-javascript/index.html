Common examples

Email shape: ^[^\s@]+@[^\s@]+\.[^\s@]+$
URL slug: ^[a-z0-9]+(?:-[a-z0-9]+)*$
Hex color: ^#(?:[0-9a-fA-F]{3}){1,2}$
Integer or decimal: ^-?\d+(?:\.\d+)?$
Collapse whitespace: \s+

How to test them safely

  • Use realistic valid and invalid input, not only happy-path samples.
  • Toggle the exact flags your JavaScript code will use.
  • Check whether the pattern should match the whole string with ^ and $.
  • Prefer simple regex plus clear code when validation rules become hard to read.

Where regex is a good fit

Regex is useful for quick structural checks, extraction, replacement, and cleanup. It is a weak fit for complex business rules, security-sensitive parsing, or validation that depends on external state.
