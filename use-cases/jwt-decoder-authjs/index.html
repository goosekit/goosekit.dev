Auth debugging

JWT decoder for Auth.js and API token debugging

Auth bugs often hide inside token claims: wrong audience, expired token, missing role, unexpected issuer, or a stale session value.

How to use this workflow

  1. Paste a JWT from a cookie, Authorization header, or test request.
  2. Read the header and payload without changing the token.
  3. Check common claims like exp, iat, aud, iss, sub, role, and provider-specific metadata.
  4. Use the timestamp converter for Unix expiry values and JSON formatter for copied claims.

When this helps

  • Debugging Auth.js session tokens
  • Checking Supabase JWT role claims
  • Inspecting API tokens before testing a protected endpoint

Quick answers

Does decoding verify the signature?

No. Decoding helps inspect token contents; signature verification requires the appropriate secret or public key.

Is it safe to paste real JWTs?

Prefer short-lived development tokens or sanitized examples. Never share production tokens in public threads or bug reports.

Why does my token look valid but fail?

Check expiry, audience, issuer, and whether the backend expects a different signing key or algorithm.
