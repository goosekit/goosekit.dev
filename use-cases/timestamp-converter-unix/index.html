Time debugging

Unix timestamp converter for logs and tokens

Unix timestamps show up in JWT claims, logs, API payloads, cron jobs, and database records, but raw numbers make time-zone mistakes easy.

How to use this workflow

  1. Copy the timestamp from your log, token, or payload.
  2. Convert it to a readable date and check the time zone assumptions.
  3. For JWTs, compare exp and iat values against the current time.
  4. For scheduling bugs, verify whether the system expects UTC or local time.

When this helps

  • Checking JWT expiration
  • Reading server logs
  • Debugging scheduled jobs and webhook timestamps

Quick answers

Seconds or milliseconds?

Many APIs use seconds, while JavaScript Date values often use milliseconds. If the date looks wildly wrong, check the unit.

Why is the time off by one or two hours?

Usually time zone or daylight-saving assumptions. Store UTC and display local time deliberately.

Can timestamps prove event order?

They help, but distributed systems can have clock drift. Pair timestamps with IDs or logs when possible.
