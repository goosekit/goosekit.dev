A narrow helper for SaaS builders who already have Stripe events and Supabase access state. Generate a SQL skeleton for the four money states that matter: active in Stripe but inactive locally, inactive in Stripe but still unlocked, missing customer links, and stale local entitlements.

The output is deliberately practical. Paste the query into Supabase SQL editor, adapt table names, then use the result as the evidence packet for a €99-149 billing drift review if it finds real mismatches.
