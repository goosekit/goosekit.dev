1. Competitor A launched “CRM for consultants”

New title, H1, and pricing block appeared this week. The page targets a clearer buyer than their homepage.

Action: publish “TinyDev CRM for solo consultants” with 3 concrete workflow examples and a migration CTA.
2. Competitor B removed the free plan from nav

Their pricing page now leads with a $15/mo starter plan and a “simple setup” promise.

Action: update your pricing FAQ around “free vs paid” and test a one-time setup helper CTA.
3. Two new directory listings mention “no setup call”

Recent listings are repeating a low-friction sales angle. That may be converting better than feature-heavy copy.

Action: add “start without a setup call” near your hero CTA and measure clicks for 7 days.
4. Reddit thread asks for “CRM that is not bloated”

The thread contains exact objections: too many fields, slow onboarding, expensive seats.

Action: create a short “not bloated CRM” proof page, then answer only if it genuinely matches the thread.

Recommended action plan for this week

Day 1: publish the consultant landing page. Use one clear segment and avoid generic startup language.

Day 2: add one comparison section: “TinyDev CRM vs spreadsheet vs bloated CRM”.

Day 3: update pricing FAQ with the free-plan objection and a low-friction starter explanation.

Day 4: monitor the Reddit thread and reply only if the proof page directly helps.

Day 7: check impressions/clicks for the new page and decide whether to create “CRM for freelancers”.

Why this report is useful

The value is not “we detected changes”. The value is knowing which small move to ship next: one page, one CTA, one reply, one pricing clarification.
