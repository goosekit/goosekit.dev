Debug packet for a paid review

If the score exposes risk, the fastest first pass is a small incident packet. It should show one real mismatch, not a full codebase tour.

  • Stripe object ids: customer, subscription, checkout session, invoice, or event id.
  • Supabase rows or table names that store user, team, subscription, entitlement, credit, or booking state.
  • Checkout Session ledger row before/after fulfillment, plus the mock fallback or Stripe env mode evidence.
  • Plan, credit-pack, or gift grant row after success-route refresh and webhook replay.
  • Entitlement/access row before and after replay, plus any tenant/workspace provisioning job or result row.
  • One-time add-credit Checkout or payment event, credit ledger/balance row, and grant function/webhook log.
  • 23505 duplicate-key log or processed-event conflict, plus the HTTP response Stripe saw after duplicate delivery.
  • For Stripe Connect: transfer id, payout balance row, commission row, clawback queue row, and the replay/idempotency result.
  • What the paid user currently sees: blocked, free plan, missing credits, duplicate access, wrong team, or stale portal state.
  • Webhook delivery status: delivered, failed, retried, duplicated, missing handler, or unknown.
  • The one state that must be true after review: paid users unlocked, unpaid users blocked, no duplicate grant, or recurring reconciliation in place.
