URL Encoding and Decoding Explained — Complete Guide (2026)

Have you ever copied a URL from your browser's address bar and ended up with a string full of %20, %3D, and %26 characters? Or pasted a URL into your code only to have it break because of spaces, ampersands, or non-ASCII characters? That is URL encoding — also known as percent-encoding — at work.

URL encoding is a foundational web technology that ensures data can be safely transmitted in URLs, HTML forms, cookies, and HTTP headers. Despite being specified in 1994 (RFC 1738, updated by RFC 3986), it remains a constant source of subtle bugs for developers who do not understand when to encode, what to encode, and which tool to use.

This guide covers everything: the specification, character-by-character rules, code examples in every major language, common mistakes that cause real outages, and best practices for 2026.

What Is URL Encoding?

URL encoding replaces unsafe ASCII characters with a % followed by two hexadecimal digits representing the character's ASCII value. For example:

"Hello World! 100% free"
 → "Hello%20World%21%20100%25%20free"

Each space ( , ASCII 32 = 0x20) becomes %20, the exclamation mark (!, ASCII 33 = 0x21) becomes %21, and the percent sign (%, ASCII 37 = 0x25) becomes %25.

Non-ASCII characters are first encoded as UTF-8 bytes, then each byte is percent-encoded:

"café" in UTF-8 bytes: 63 61 66 C3 A9
 → "caf%C3%A9"

"日本語" in UTF-8 bytes: E6 97 A5 E6 9C AC E8 AA 9E
 → "%E6%97%A5%E6%9C%AC%E8%AA%9E"

Which Characters Need Encoding?

According to RFC 3986, a URL is composed of unreserved characters, reserved characters, and everything else:

Unreserved Characters (NEVER encode these)

A–Z  a–z  0–9  -  _  .  ~

These 66 characters are safe in any part of a URL and should never be percent-encoded. Encoding them (e.g., turning a hyphen into %2D) is technically valid but produces unnecessarily long and hard-to-read URLs.

Reserved Characters (MUST encode in data context)

Character Encoded Reserved Purpose
!%21Delimiter
#%23Fragment identifier
$%24Delimiter
&%26Query parameter separator
'%27Delimiter
(%28Delimiter
)%29Delimiter

The rule of thumb: if a character has special meaning in a URL context (separating path segments, delimiting query parameters, etc.), you must percent-encode it when that character appears as data rather than as a structural element.

Practical Examples

Query Parameter Encoding

// Building a search URL with special characters
const query = "cats & dogs: 100% fun!";
const encoded = encodeURIComponent(query);
console.log(encoded);
// "cats%20%26%20dogs%3A%20100%25%20fun!"

const url = `https://example.com/search?q=${encoded}`;
console.log(url);
// "https://example.com/search?q=cats%20%26%20dogs%3A%20100%25%20fun!"

Path Segment Encoding

// File paths with special characters
const filename = "report Q1 2026 (final).pdf";
const encoded = encodeURIComponent(filename);
const url = `https://goosekit.dev/files/${encoded}`;
// https://goosekit.dev/files/report%20Q1%202026%20(final).pdf

JSON in Query Parameters

// Sending structured data as a query parameter
const filter = { status: "active", tags: ["api", "webhook"] };
const json = JSON.stringify(filter);
const encoded = encodeURIComponent(json);
const url = `https://api.example.com/orders?filter=${encoded}`;
// https://api.example.com/orders?filter=%7B%22status%22%3A%22active%22%2C%22tags%22%3A%5B%22api%22%2C%22webhook%22%5D%7D

URL Encoding in Code

JavaScript (Browser and Node.js)

// Built-in functions (available everywhere)
const text = "Hello, World! café";

// Encode — encodes everything except unreserved chars
const encoded = encodeURIComponent(text);
console.log(encoded);
// Hello%2C%20World!%20caf%C3%A9

// Decode
const decoded = decodeURIComponent(encoded);
console.log(decoded); // Hello, World! café

// ⚠️ DON'T use encodeURI() for query parameter values
// encodeURI() does NOT encode: # $ & + , / : ; = ? @
// These are structural characters that MUST be encoded in data
const uri = "https://example.com/path with spaces";
const safeUri = encodeURI(uri); // Encodes spaces but keeps structure

// ⚠️ decodeURIComponent throws on malformed input
try {
  decodeURIComponent("%E0%A4%A");
} catch (e) {
  console.error("Malformed URI:", e.message);
  // URIError: URI malformed
}

Python

from urllib.parse import quote, unquote, quote_plus, urlencode

text = "Hello, World! café"

# encodeURIComponent equivalent
encoded = quote(text, safe="")
print(encoded)  # Hello%2C%20World!%20caf%C3%A9

# decodeURIComponent equivalent
decoded = unquote(encoded)
print(decoded)  # Hello, World! café

# application/x-www-form-urlencoded (spaces → +)
form_encoded = quote_plus(text)
print(form_encoded)  # Hello%2C+World%21+caf%C3%A9

# Build query strings
params = {"q": "search query", "page": 2, "lang": "en"}
query_string = urlencode(params)
print(query_string)  # q=search+query&page=2&lang=en

Go (Golang)

package main

import (
    "fmt"
    "net/url"
)

func main() {
    text := "Hello, World! café"

    // Encode
    encoded := url.QueryEscape(text)
    fmt.Println(encoded) // Hello%2C+World%21+caf%C3%A9

    // Decode
    decoded, _ := url.QueryUnescape(encoded)
    fmt.Println(decoded) // Hello, World! café

    // Build a full URL safely
    u := &url.URL{
        Scheme:   "https",
        Host:     "api.example.com",
        Path:     "/v1/search",
    }
    q := u.Query()
    q.Set("q", text)
    q.Set("limit", "10")
    u.RawQuery = q.Encode()
    fmt.Println(u.String())
    // https://api.example.com/v1/search?q=Hello%2C+World%21+caf%C3%A9&limit=10
}

Java

import java.net.URLEncoder;
import java.net.URLDecoder;
import java.nio.charset.StandardCharsets;

public class UrlEncoding {
    public static void main(String[] args) throws Exception {
        String text = "Hello, World! café";

        // Encode (Java 10+)
        String encoded = URLEncoder.encode(text, StandardCharsets.UTF_8);
        System.out.println(encoded); // Hello%2C+World%21+caf%C3%A9

        // Decode
        String decoded = URLDecoder.decode(encoded, StandardCharsets.UTF_8);
        System.out.println(decoded); // Hello, World! café
    }
}

Common Pitfalls and How to Avoid Them

1. Using encodeURI() Instead of encodeURIComponent()

// ❌ Wrong — preserves structural characters that should be encoded in values
const bad = encodeURI("key=value&other=data");
// "key=value&other=data" — & and = are NOT encoded!

// ✅ Correct — encodes everything except unreserved characters
const good = encodeURIComponent("key=value&other=data");
// "key%3Dvalue%26other%3Ddata"

Use encodeURI() only for complete URL strings where structural characters must be preserved. Use encodeURIComponent() for individual URL components (query values, path segments, fragment identifiers).

2. Mixing + and %20 for Spaces

The history is confusing: in application/x-www-form-urlencoded (HTML form submissions), spaces are encoded as +. In application/x-www-form-urlencoded query strings, browsers accept both. But + is a literal plus sign in standard percent-encoding. When in doubt, use %20 for spaces.

// In query strings: both are generally accepted
"?q=hello+world"  // + means space (form encoding)
"?q=hello%20world" // %20 means space (percent-encoding)

// But in path segments: + is literal
"/path/hello+world"  // The + is part of the path, NOT a space
"/path/hello%20world" // This IS a space in the path

3. Double Encoding

Encoding already-encoded data creates strings that look like %2520 instead of %20. This happens when a framework automatically encodes and you also encode manually:

// ❌ Double encoding
const once = encodeURIComponent("hello world");
// "hello%20world"
const twice = encodeURIComponent(once);
// "hello%2520world" ← broken!

// ✅ Encode once, at the right level
const params = new URLSearchParams({ q: "hello world" });
const url = `/search?${params.toString()}`;
// "/search?q=hello+world"

4. URL Fragment Identifiers (#)

The # character and everything after it is never sent to the server — it is handled entirely client-side. If you need to include a hash in a value, encode it:

// ❌ The # starts a fragment — "section1" is never sent to the server
fetch("/api/article?content=Intro#section1")

// ✅ Properly encode the #
encodeURIComponent("Intro#section1")
// "Intro%23section1"

URL Encoding of JSON (Advanced)

Sometimes you need to embed JSON directly in a URL. This is common for sharing filters, search parameters, or configuration state:

const filters = {
  category: "api-tools",
  tags: ["free", "no-signup"],
  sort: "-created_at",
  page: 1
};

const json = JSON.stringify(filters);
const encoded = encodeURIComponent(json);
const shareUrl = `https://goosekit.dev/tools?filters=${encoded}`;

// shareUrl:
// https://goosekit.dev/tools?filters=%7B%22category%22%3A%22api-tools%22%2C%22tags%22%3A%5B%22free%22%2C%22no-signup%22%5D%2C%22sort%22%3A%22-created_at%22%2C%22page%22%3A1%7D

For the receiving side:

// Parse the URL parameter back to an object
const params = new URLSearchParams(window.location.search);
const filters = JSON.parse(params.get("filters"));

Conclusion

URL encoding is simple once you internalize two rules: encode structural characters when they appear as data, and always use UTF-8 as the underlying character encoding. The most common bugs — double encoding, encodeURI vs encodeURIComponent confusion, and + vs %20 ambiguity — all stem from not understanding where in the URL a string is being used.

When in doubt, run your strings through a tool to verify the encoding is correct before shipping.
