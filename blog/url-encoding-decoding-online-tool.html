What Is URL Encoding?

URL encoding — formally known as percent encoding — is a mechanism used to translate characters that are not allowed or have special meaning in a Uniform Resource Locator (URL) into a universally accepted format. Every developer working with web applications, APIs, or even crafting a simple query string encounters URL encoding daily.

URLs can only contain a restricted set of ASCII characters: digits, letters, and a handful of symbols like hyphens (-), underscores (_), periods (.), and tildes (~). Everything else — spaces, ampersands, equals signs, non-ASCII characters like é or 日本語 — must be encoded.

How Percent Encoding Works

The encoding process is straightforward: each unsafe character is replaced by a percent sign (%) followed by the character's two-digit hexadecimal ASCII value. For example:

  • A space ( ) becomes %20
  • An ampersand (&) becomes %26
  • A question mark (?) becomes %3F

UTF-8 multibyte characters get each byte encoded separately. The emoji 🔥 (U+1F525) becomes %F0%9F%94%A5 — four percent-encoded bytes.

Common Scenarios Where URL Encoding Matters

1. Building Query Strings

When constructing URLs with parameters, you must encode both the key and the value. Consider this common mistake:

// ❌ BAD — spaces and special characters break the URL
const url = "https://api.example.com/search?q=hello world&category=art&design";

// ✅ GOOD — properly encoded
const url = "https://api.example.com/search?q=hello%20world&category=art%26design";

The unencoded version would cause the & in "art&design" to be interpreted as a parameter separator, splitting your category value in half. This is one of the most frequent bugs in web development.

2. Sending API Requests

REST APIs expect properly encoded parameters. Here's how you'd safely build a query string in JavaScript using the built-in URLSearchParams API:

const params = new URLSearchParams({
  query: 'café & gâteau',
  limit: 25,
  filter: 'status:active'
});

console.log(params.toString());
// Output: query=caf%C3%A9+%26+g%C3%A2teau&limit=25&filter=status%3Aactive

const url = `https://api.example.com/v2/items?${params.toString()}`;
console.log(url);
// https://api.example.com/v2/items?query=caf%C3%A9+%26+g%C3%A2teau&limit=25&filter=status%3Aactive

Notice how URLSearchParams automatically handles spaces (as + in form encoding), accented characters, and colons. But if you're building URLs manually — say in a configuration file or templating system — you need to encode yourself.

3. Embedding URLs Inside Other URLs

Redirect URLs and callback URLs in OAuth flows are a classic example:

const redirectUrl = 'https://myapp.com/callback?from=github&action=login';
const encoded = encodeURIComponent(redirectUrl);
console.log(encoded);
// https%3A%2F%2Fmyapp.com%2Fcallback%3Ffrom%3Dgithub%26action%3Dlogin

// Now it can be safely nested
const oauthUrl = `https://github.com/login/oauth/authorize?client_id=abc&redirect_uri=${encoded}`;

Without encoding the colon, slashes, and question marks in your redirect URL, the outer URL structure would break entirely.

encodeURI vs encodeURIComponent — Know the Difference

JavaScript provides two encoding functions, and choosing the wrong one is a common source of bugs. Here's the comparison developers need to keep in mind:

FunctionEncodesUse Case
encodeURI()Non-ASCII charactersEncoding a full URL
encodeURIComponent()Almost everythingEncoding a URL component (query value, path segment)
 
const url = "https://api.example.com/search?q=hello world&page=2";

encodeURI(url)
// "https://api.example.com/search?q=hello%20world&page=2"
// → Keeps : / ? & intact (valid URL structure characters)

encodeURIComponent(url)
// "https%3A%2F%2Fapi.example.com%2Fsearch%3Fq%3Dhello%20world%26page%3D2"
// → Encodes EVERYTHING, including protocol and separators

Rule of thumb: use encodeURI() on complete URLs, and encodeURIComponent() on the pieces you insert into a URL.

Decoding URL-Encoded Strings

Decoding reverses the process, converting %XX sequences back to their original characters. This is essential when reading URL parameters on the server side or parsing API responses.

const encoded = "search%3Fq%3Dcaf%C3%A9+%26+g%C3%A2teau";

// Using decodeURIComponent
console.log(decodeURIComponent(encoded));
// "search?q=café & gâteau"

URL Encoding vs HTML Encoding

Don't confuse URL encoding (percent encoding) with HTML entity encoding. They serve different purposes:

  • URL encoding: < → %3C — used in URLs, query strings, and paths.
  • HTML encoding: < → &lt; — used to prevent XSS in HTML content.

Using the wrong encoding in the wrong context is a common mistake that can lead to broken links or security vulnerabilities. Always match the encoding to the context in which the data will be rendered or transmitted.

Encoding in Other Languages

While JavaScript's built-in functions are convenient, you'll encounter URL encoding in every language. Here's how to encode and decode URLs across popular programming environments:

# Python
from urllib.parse import quote, unquote

encoded = quote("café & gâteau", safe="")
# caf%C3%A9%20%26%20g%C3%A2teau

decoded = unquote(encoded)
# café & gâteau
# Ruby
require 'uri'

encoded = URI.encode_www_form_component("café & gâteau")
decoded = URI.decode_www_form_component(encoded)
# cURL (command line)
curl --data-urlencode "q=café & gâteau" https://api.example.com/search
// PHP
$encoded = urlencode("café & gâteau");  // caf%C3%A9+%26+g%C3%A2teau
$decoded = urldecode($encoded);         // café & gâteau
