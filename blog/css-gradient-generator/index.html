What Is a CSS Gradient?

A CSS gradient is a method of creating a smooth transition between two or more colors using pure CSS — no images required. The browser renders gradients on the fly as part of the painting step in the rendering pipeline, which means they scale infinitely, work at any resolution, and have a tiny footprint compared to gradient images saved as PNGs or JPEGs.

CSS gradients are technically images generated by the browser. They can be used anywhere a URL to an image would go — background-image, border-image, mask-image, list-style-image, and even as pseudo-element content. There are three types of gradients in CSS, each producing a different visual effect.

Understanding gradients means understanding how color stops work. A color stop defines where a particular color appears along the gradient line. The browser interpolates smoothly between stops. With only two stops (at 0% and 100%), the transition is even across the entire gradient. But adding intermediate stops gives you full control — you can create bands of color, abrupt transitions, or subtle multi-step blends.

The gradient specification has evolved significantly over the years. Modern syntax is supported in all major browsers without vendor prefixes, but knowing how fallbacks and legacy syntax work is still useful for maintaining older projects.

Linear Gradients

A linear gradient transitions colors along a straight line. You control the direction (or angle) and the color stops. This is the most commonly used gradient type — you will find it on hero sections, card backgrounds, button states, and call-to-action banners.

Here is the basic syntax:

/* Simple top-to-bottom gradient */
background: linear-gradient(to bottom, #6366f1, #ec4899);

/* Diagonal gradient */
background: linear-gradient(135deg, #0ea5e9, #8b5cf6);

/* Multi-stop gradient with custom positions */
background: linear-gradient(
  90deg,
  #f59e0b 0%,
  #ef4444 33%,
  #8b5cf6 66%,
  #06b6d4 100%
);

The to <side> syntax is the most readable — to bottom right, to left, to top. Or you can use angles in degrees where 0deg points upward and angles increase clockwise.

Radial Gradients

A radial gradient radiates outward from a central point. It is perfect for creating spotlight effects, glowing orbs, subtle vignettes, or backgrounds that draw attention toward the center of the viewport.

/* Simple radial from center */
background: radial-gradient(circle, #6366f1, #1e1b4b);

/* Elliptical radial, positioned off-center */
background: radial-gradient(ellipse at 30% 20%, #f59e0b, #7c2d12);

/* Multi-stop radial with hard edges */
background: radial-gradient(
  circle closest-side,
  #f472b6 0%,
  #f472b6 30%,
  #8b5cf6 30%,
  #8b5cf6 60%,
  #06b6d4 60%,
  #06b6d4 100%
);

The shape keyword controls the overall form: circle for a perfect circle, ellipse for a stretched oval. Position keywords like at center, at top right, or percentage values place the origin point. Size keywords (closest-side, farthest-corner, contain, cover) determine how far the gradient extends.

Conic Gradients

A conic gradient sweeps colors around a center point like a pie chart. This is the newest gradient type in CSS and is perfect for creating color wheels, radar-like backgrounds, pie charts, and circular loading indicators.

/* Simple conic with three colors */
background: conic-gradient(#ef4444, #22c55e, #3b82f6, #ef4444);

/* Conic starting from a specific angle */
background: conic-gradient(from 45deg, #f59e0b, #ec4899, #8b5cf6, #f59e0b);

/* Conic pie chart with hard stops */
background: conic-gradient(
  from 0deg,
  #22c55e 0deg 120deg,
  #3b82f6 120deg 240deg,
  #f59e0b 240deg 360deg
);

The from keyword sets the starting angle. Using degree-based color stops instead of percentages gives you precise control over each segment — ideal for data visualization.

Advanced Gradient Techniques

Once you are comfortable with the basics, these techniques take your gradients to the next level.

Transparent Overlays

One of the most powerful patterns is placing a gradient with alpha transparency over a background image. This creates a dramatic darkening or color-tinting effect that makes overlaid text highly readable:

.hero {
  background:
    linear-gradient(
      to right,
      rgba(15, 23, 42, 0.85),
      rgba(15, 23, 42, 0.3)
    ),
    url('hero-photo.jpg');
  background-size: cover;
  background-position: center;
}

The key insight: background-image accepts a comma-separated list. The first layer renders on top, so the gradient overlays the image. You can combine multiple gradients this way, too.

Animated Gradients

CSS animations on gradient backgrounds are surprisingly performant when you animate the background-position or use a pseudo-element with transform:

.animated-bg {
  background: linear-gradient(
    270deg,
    #6366f1, #ec4899, #f59e0b, #22c55e, #6366f1
  );
  background-size: 400% 400%;
  animation: gradientShift 12s ease infinite;
}

@keyframes gradientShift {
  0%   { background-position: 0% 50%; }
  50%  { background-position: 100% 50%; }
  100% { background-position: 0% 50%; }
}

Making the background-size larger than the element and animating background-position creates a smooth, flowing color shift. This technique is used on many high-end landing pages and costs virtually nothing in terms of GPU usage compared to JavaScript-based animation.

Border Gradients

Before border-image had broad support, gradient borders required tricky pseudo-elements. Now you can use border-image or the background-clip trick for rounded borders:

/* Gradient border on a rounded card */
.gradient-border {
  border-radius: 16px;
  background: linear-gradient(#1e1b4b, #1e1b4b) padding-box,
              linear-gradient(135deg, #818cf8, #c084fc) border-box;
  border: 2px solid transparent;
  padding: 24px;
}

The padding-box / border-box layering technique is the most reliable way to get gradient borders with arbitrary border-radius — it works in every modern browser.

Text Gradients

CSS text gradients use background-clip: text with a transparent text color:

.gradient-text {
  font-size: 3rem;
  font-weight: 800;
  background: linear-gradient(90deg, #6366f1, #ec4899);
  background-clip: text;
  -webkit-background-clip: text;
  color: transparent;
}

The -webkit- prefix is still needed for Chromium-based browsers as of writing — background-clip: text remains a prefixed feature despite broad adoption.

Gradients in Popular CSS Frameworks

If you use a framework, gradients are often built in as utility classes. Here is how to use and customize them.

Tailwind CSS

<!-- Horizontal gradient utility -->
<div class="bg-gradient-to-r from-indigo-500 via-purple-500 to-pink-500"></div>

<!-- Radial (requires tailwind.config.js extension) -->
<!-- Add to config: theme.extend.backgroundImage['radial-gradient'] = 'radial-gradient(var(--tw-gradient-stops))' -->
<div class="bg-radial-gradient from-blue-400 to-slate-900"></div>

Tailwind includes 33 gradient utilities out of the box (combinations of bg-gradient-to-{direction} and the from-, via-, to- color classes). For radial or conic gradients, you need to extend the config — or just write the custom CSS directly.

Bootstrap 5

<!-- Bootstrap does not ship gradient utilities by default -->
<!-- Use inline style or a custom class -->
<div style="background: linear-gradient(135deg, #0d6efd, #6610f2);"></div>

Bootstrap takes a lightweight approach and provides minimal gradient utilities. The recommended pattern is to define your gradients as CSS custom properties in a _gradients.scss partial and include them via a single class.

Vanilla CSS Custom Properties

:root {
  --gradient-primary: linear-gradient(135deg, oklch(60% 0.25 260), oklch(60% 0.25 320));
  --gradient-hero: linear-gradient(180deg, #0c0a09 0%, #1c1917 50%, #292524 100%);
}

.card { background: var(--gradient-primary); }
.hero-section { background: var(--gradient-hero); }

Using oklch() in CSS custom properties produces gradients with perceptually uniform color transitions — no more muddy or over-saturated midpoints. This is the modern best practice.
