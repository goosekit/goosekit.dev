How to Generate Secure Passwords — Complete Guide (2026)

Every single day, millions of credentials are exposed in data breaches. The 2025 Verizon Data Breach Investigations Report found that over 80% of hacking-related breaches involved weak, stolen, or default passwords. If you are still using variations of your pet's name with a number appended, you are playing defense with the door wide open.

This guide covers everything you need to know about generating strong passwords — from the mathematics behind password entropy to practical code you can run right now. Whether you are a developer building authentication systems or an everyday user who wants better security, this article has you covered.

What Makes a Password "Strong"?

Password strength is measured in entropy — expressed in bits. Higher entropy means more possible combinations, which makes brute-force attacks exponentially harder. Each additional character from a larger character set adds significantly to your security.

The formula for password entropy is straightforward:

entropy = length × log₂(character_set_size)

So a 12-character password using lowercase letters only (26 characters) gives us:

entropy = 12 × log₂(26) ≈ 12 × 4.7 ≈ 56.4 bits

That same 12 characters using a mixed set of uppercase, lowercase, digits, and symbols (94 characters) yields:

entropy = 12 × log₂(94) ≈ 12 × 6.55 ≈ 78.6 bits

A 22-bit difference means the mixed-set password is roughly 4 million times harder to crack through brute force. Length matters, but character diversity matters just as much.

Common Password Mistakes to Avoid

  1. Using personal information. Dog names, birthdays, and street addresses are trivially guessable or discoverable through social media. Attackers run dictionaries that include names pulled from your own Facebook profile.
  2. Simple character substitutions. Replacing "a" with "@", "e" with "3", or "o" with "0" does not fool modern cracking tools. Hashcat and John the Ripper include these as standard rules in every dictionary attack.
  3. Reusing passwords across sites. If one site gets breached and you used the same password everywhere, every other account is compromised. Credential stuffing attacks test leaked passwords against hundreds of services automatically.
  4. Short passwords on "important" accounts. Your email account is the master key to everything else. An attacker who controls your email can reset passwords for every other service. Give it a truly strong, unique password.
  5. Writing passwords on sticky notes. Physical security matters too. An unlocked workstation with a visible sticky note defeats any cryptographic strength.

How to Generate Secure Passwords Programmatically

If you are building an application or just want to generate passwords from the command line, here are production-quality implementations in the most common environments.

Python: Cryptographically Secure Password Generation

Python's secrets module generates cryptographically strong random numbers. Never use random for passwords — its Mersenne Twister is not designed for security.

import secrets
import string

def generate_password(length=16):
    """Generate a cryptographically secure random password."""
    alphabet = string.ascii_letters + string.digits + "!@#$%^&*"
    while True:
        password = ''.join(secrets.choice(alphabet) for _ in range(length))
        # Ensure at least one of each character type
        if (any(c.islower() for c in password) and
            any(c.isupper() for c in password) and
            any(c.isdigit() for c in password) and
            any(c in "!@#$%^&*" for c in password)):
            return password

print(generate_password(20))
# Example: kR7$mN2#pL9@xQ4!wZ8v

Node.js / JavaScript

Using the built-in crypto module for secure random bytes:

const crypto = require('crypto');

function generatePassword(length = 16) {
  const charset = 'abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789!@#$%^&*';
  let password = '';
  const bytes = new Uint32Array(length);
  crypto.randomFillSync(bytes);

  for (let i = 0; i < length; i++) {
    password += charset[bytes[i] % charset.length];
  }
  return password;
}

console.log(generatePassword(20));
// Example: T5n$kW8@pM2#rJ7!xQ4v

Command Line with OpenSSL

Need a quick password from your terminal? OpenSSL's pseudo-random generator is more than adequate:

# Generate a 24-character base64 password
openssl rand -base64 24
# Example: 7kR9mN2pL5xQ8wZ4vT3yU1@#

# Generate a 20-character hex password
openssl rand -hex 20
# Example: a3f8c2d9e1b7f4a6c8d2e5f1a9b3c7d0e4f6a8b2

# Generate a password with specific charset (bash)
cat /dev/urandom | tr -dc 'a-zA-Z0-9!@#$%^&*()' | fold -w 16 | head -n 1

Bash with /dev/urandom

# Generate a 32-character password from /dev/urandom
head /dev/urandom | tr -dc A-Za-z0-9'!@#$%^&*' | head -c 32; echo

# Generate a memorable passphrase from a word list
shuf -n 4 /usr/share/dict/words | tr '\n' '-' | sed 's/-$//'

The Dice-Passphrase Method (Diceware)

For passwords you actually need to remember — your master password, for instance — the Diceware method is the gold standard. It combines strong entropy with human memorability.

How it works:

  1. Roll a physical die 5 times to generate a 5-digit number (e.g., 43216).
  2. Look up that number in the official Diceware word list.
  3. Repeat 6–8 times.
  4. Concatenate the words with separators.

Each word from the official Diceware list contributes approximately 12.9 bits of entropy (log₂(7776) since there are 7,776 words). A 7-word passphrase gives roughly 90 bits — comfortably out of reach of brute-force attacks for the foreseeable future.

Example: "correct horse battery staple rename twilight"
7 words × 12.9 bits = 90.3 bits of entropy
Estimated crack time at 10 billion guesses/sec: ~4 million years

Tools that claim to generate "memorable passwords" by replacing letters with numbers (P@ssw0rd) are fundamentally broken. Real memorability comes from random word sequences, not obfuscation.

Password Storage: A Developer's Responsibility

If you are storing user passwords server-side, generating strong passwords is only half the story. You must hash them properly. Here is what a secure password storage implementation looks like:

Proper Password Hashing in Python

from argon2 import PasswordHasher

ph = PasswordHasher(
    time_cost=3,          # Number of iterations
    memory_cost=65536,    # 64 MB memory
    parallelism=4,        # Parallel threads
    hash_len=32,          # Output hash length
    salt_len=16           # Random salt length
)

# Hash a new password
hashed = ph.hash("user_secure_password_here")

# Verify a login attempt
try:
    ph.verify(hashed, "user_secure_password_here")
    print("Password is correct!")
except argon2.exceptions.VerifyMismatchError:
    print("Password is incorrect!")

Argon2id (winner of the Password Hashing Competition) is the recommended algorithm in 2026. Alternatives in order of preference:

  • Argon2id — Best overall (memory-hard, side-channel resistant)
  • Bcrypt — Battle-tested, widely available, good choice
  • Scrypt — Good but less widely supported
  • PBKDF2-SHA256 — Acceptable but not preferred (NIST still approves it)
  • MD5 / SHA1 / SHA256 (plain) — NEVER use these for passwords

Multi-Factor Authentication: The Final Layer

Even the strongest password can be compromised through phishing, keyloggers, or social engineering. Multi-factor authentication (MFA) adds a second layer that an attacker must bypass. In 2026, your options are:

  • Hardware security keys (FIDO2/WebAuthn) — The strongest option. YubiKey and similar devices that authenticate via USB/NFC/Bluetooth. Phishing-resistant by design.
  • Authenticator apps (TOTP) — Google Authenticator, Authy, or your password manager's built-in OTP generator. Far better than SMS.
  • Passkeys — The emerging standard that replaces passwords entirely with cryptographic key pairs synced via your device platform.
  • SMS codes — Better than nothing, but vulnerable to SIM-swapping attacks. Use only when nothing else is available.

UUIDs and Random Tokens

Sometimes you don't need a password — you need a unique, unpredictable identifier for API keys, session tokens, or reset links. UUIDs (Universally Unique Identifiers) serve this purpose well.

# UUID version 4 (random) — 122 bits of randomness
import uuid; print(uuid.uuid4())
# Example: 9b1deb4d-3b7d-4bad-9bdd-2b0d7b3dcb6d

# Secure random hex token (256 bits)

Password Managers: Why You Need One

The most practical way to manage dozens of unique, strong passwords is a dedicated password manager. Here is why:

  • You only need to remember one master password. Make that one truly strong (20+ characters) and your entire password vault is protected.
  • Auto-fill protects against phishing. A password manager will only fill credentials on the exact domain where they were saved. If you land on g00gle.com, it won't fill your Google password.
  • Built-in generators. Most password managers include generators that create strong passwords as you sign up for new services.
  • Secure sharing. Some managers allow encrypted sharing of credentials with family members or team members.

Whether you choose Bitwarden, 1Password, KeePass, or your browser's built-in manager, the important thing is to use one rather than recycling passwords.

Common Questions About Password Security

How long should a password be in 2026?

The absolute minimum is 12 characters for non-critical accounts and 16+ for anything important (email, banking, password manager). If you are using a passphrase (random word sequence), aim for 6–8 words.

Should I change my passwords regularly?

NIST and Microsoft both recommend against mandatory periodic password changes unless there is evidence of compromise. Frequent forced changes lead to weaker passwords (users pick simpler ones or increment numbers). Change only when you suspect unauthorized access or a service has been breached.

Are password generators safe to use?

Generators that run entirely in your browser (like Goosekit's Password Generator) are safe because the randomness happens locally and the password never traverses the network. Avoid any online generator that sends your password to a server.
