The exact stack: Next.js 15 (App Router), Supabase auth + database, Stripe Checkout + Customer Portal + webhook handlers, React Email templates, Tailwind CSS + Shadcn/ui, landing page, dashboard scaffold, settings and billing pages.

2. How Fast Is It to Get Running?

The README takes you through the full setup in order. If you already have a Supabase project and Stripe account ready, getting to a live local environment in under 30 minutes is a realistic target for many builders.

The things that still require your attention after clone:

3. Does It Work Well With AI Coding Tools?

This was the main design goal. The file structure is intentionally flat and annotated. There are no complex dependency webs or magic wrappers to understand.

When you're prompting Cursor, Claude, or Copilot to build your product, you want them working on your features — not trying to understand why the auth middleware is 400 lines of abstraction.

Every file has a clear purpose. The README calls out which files handle what. I used this myself while building with AI tools and it noticeably reduced bad generations compared to messier starters I've used.
