Encode the full URL or only a component?

This is the mistake that causes most confusion. If you encode a whole URL with encodeURIComponent(), it will also encode slashes, colons, and other characters that should remain intact in the overall URL structure. If you use encodeURI() on a single query parameter, it will not escape enough characters.

The safe mental model is simple:

  • Use encodeURI() when you already have a complete URL and want to preserve its structure.
  • Use encodeURIComponent() when you are encoding one value inside a URL, such as a query parameter.

A practical JavaScript example

Here is the pattern most frontend and Node developers end up using:

const base = 'https://example.com/search';
const query = 'c# url encoding & decoding';
const redirect = 'https://app.example.com/callback?tab=profile';

const url = `${base}?q=${encodeURIComponent(query)}&redirect=${encodeURIComponent(redirect)}`;

console.log(url);
// https://example.com/search?q=c%23%20url%20encoding%20%26%20decoding&redirect=https%3A%2F%2Fapp.example.com%2Fcallback%3Ftab%3Dprofile

console.log(decodeURIComponent('c%23%20url%20encoding%20%26%20decoding'));
// c# url encoding & decoding

Watch out for double encoding

One of the most common bugs is encoding an already encoded value. For example, a space encoded once becomes %20. If you encode that string again, the percent sign itself gets encoded and you end up with %2520. Suddenly your backend or redirect logic sees the wrong value.

A URL decoder is useful here because it lets you inspect whether the string has been encoded one time too many. If the output still looks encoded after one decode pass, that is a strong hint the value was wrapped twice somewhere upstream.

When to use URLSearchParams instead

If you are building query strings programmatically, URLSearchParams is often the cleanest approach because it handles encoding for you:

const params = new URLSearchParams({
  q: 'free online tools for developers',
  page: '1',
  ref: 'blog'
});

const url = `https://goosekit.dev/search?${params.toString()}`;
console.log(url);
// https://goosekit.dev/search?q=free+online+tools+for+developers&page=1&ref=blog

Even then, a free online encoder/decoder still helps when you are verifying output from logs, external tools, or hand-built links that did not come from your own code.
